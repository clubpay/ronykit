Use the rony/errs package for all error definitions and construction. Two primary patterns: (1) errs.GenWrap(errs.StatusCode, "ERROR_CODE") for wrappable sentinel errors that accept additional context via err("detail text"), and (2) errs.B().Code(errs.StatusCode).Msg("ERROR_CODE").Err() for static sentinel errors. Error codes are SCREAMING_SNAKE_CASE strings (e.g. REFRESH_TOKEN_EXPIRED, ACCOUNT_NOT_FOUND, SERVICE_FAILED). In API handlers, wrap errors from the app layer with errs.B().Cause(err).Msg("OPERATION_FAILED").Err(). Common status codes: errs.NotFound, errs.InvalidArgument, errs.FailedPrecondition, errs.Internal, errs.Aborted, errs.DeadlineExceeded, errs.ResourceExhausted. Place all domain-specific errors in internal/domain/errors.go as package-level variables.
